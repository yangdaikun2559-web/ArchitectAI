# ArchitectAI 物联网智能生成与编译平台 - 课件项目优化方向与实施方案 (v2.0)

本文件详细阐述了针对 **“2026年四川省教师数字素养提升实践活动常规项目（课件类）”** 的平台优化路径。本方案旨在将现有的 ArchitectAI 平台重塑为一款高教学适配性、强人机互动性、具有电气安全规则报告、模拟编译仿真、分层脚手架教学及教师诊断评价看板的**“物联网探究式学习与仿真实验课件软件”**，深度契合教师数字素养提升的评价标准。

## 二、 基础优化模块（1.0 核心框架）

### 1. 经典教学实验库 (Classic IoT Labs)
在侧边栏新增“经典实验”模块，提供中小学及职业院校物联网课程中最常用的标准实验模板：
*   **基础级**：温湿度监测系统（DHT11 + OLED）
*   **控制级**：智能路灯控制器（光敏传感器 + 继电器 + LED）
*   **报警级**：温室大棚自动化警备（AHT20 + SGP30 气体 + 有源蜂鸣器）
*   **进阶级**：智能防盗门禁（RFID 读卡器 + SG90 舵机 + LCD1602 显示）
> **教学价值**：学生无需从零打字输入需求，可以通过一键导入这些标准案例并在此基础上修改，作为探究式学习的“脚手架”。

#### 2. 元器件百科知识抽屉 (Component Wiki Drawer)
*   **交互逻辑**：在物理接线图（CAD 预览页）中，鼠标悬停或点击任意一个硬件外设（如 DHT22、继电器等）。
*   **展示内容**：展示该外设的**工作原理**、**引脚定义图**、**工作电压范围 (3.3V/5V)**、以及**通信接口协议特征**。

#### 3. 基础知识与烧录帮助中心 (Built-in Help Docs)
*   提供标准的帮助导航页，介绍**硬件连线安全守则**以及**代码烧录说明**，指导学生如何将导出的 ZIP 压缩包导入 PlatformIO 或 Keil MDK 中进行编译和硬件烧录。

### 4. 可视化电气安全检测报告 (ERC Linter Report)
*   在 CAD 预览页面下方设计一个**“物联网硬件安全检测报告”**模块，展示：
    *   🟢 **完全安全 (PASS)**：主控引脚无冲突，各总线器件通信协议配置正确。
    *   🟡 **安全警告 (WARNING)**：例如检测到 5V 继电器连接了 3.3V 的 ESP32 GPIO 端口，会弹出警告并给出硬件解决方案。
    *   🔴 **配置错误 (ERROR)**：检测到引脚强占冲突，告知学生哪两个设备的连线发生了重叠，指导其修改。

### 5. 交互式引脚拓扑微调器 (Interactive Cable Tuner)
*   实现“修改连线 $\rightarrow$ 代码宏自动改变”的双向对齐：
    1.  当学生在 CAD 连线图上面板，点击连接到主控芯片的某一根线缆时，引脚节点处弹出下拉选择框（如：PB12 切换为 PA8）。
    2.  学生点击修改引脚后，连线位置动态变化。
    3.  系统在前端同步调用 `syncFirmwarePins` 函数，在代码文件树中的 `pins.h` 中**自动覆写**对应的引脚定义（如：`#define PIN_DHT_DATA 12` 自动修改为 `#define PIN_DHT_DATA 8`）。

### 6. 在线固件语法预审 (GCC DRY-RUN 仿真)
*   在代码预览页增加“编译预审”按钮和“编译器输出终端”控制台窗口（Swiss 极简黑底绿字风格）。
*   学生点击“编译预审”，控制台开始模拟打印编译日志，检测包含关系与函数结构，如果用户在自定义代码时误删了宏定义，控制台将准确报错，增强课件的科学严谨性。

## 三、 高阶“数字素养”教育升级模块（2.0 深度优化）

为了进一步体现“以学习者为中心”的课程改革理念、培养学生的人工智能素养与教师的数字化评估能力，特增加以下深度优化：

### 1. 差异化分层教学设计（Novice to Expert 模式）
在需求输入和优化确认页，为教师提供一键切换“学生分层支持”的选项：
*   **初学模式 (Novice)**：
    *   *机制*：AI 生成完整、无缺失的代码，且配以保姆级中文逐行注释。
    *   *目标*：适合零基础学生，重在建立物理连线与驱动代码的感性认知。
*   **探究模式 (Explorer - 扣空挑战)**：
    *   *机制*：AI 会故意将核心控制函数或关键分支逻辑“扣空”（例如：留下 `// TODO: [探究任务] 请补充读取引脚并触发有源蜂鸣器报警的代码` ），并在右侧侧边栏给出逻辑流程图（Flowchart）与 API 提示。
    *   *目标*：适合有基础的学生进行课上代码填空练习，考核逻辑思维。
*   **创客模式 (Expert)**：
    *   *机制*：生成无扣空的工业级模块化 C++ 代码。
    *   *目标*：适合高级兴趣小组，重在系统集成与综合调试。

### 2. AI 智能代码解释器与互动答疑（AI 助教面板）
在代码预览页（CodeViewer）侧边，增加一个**“AI 助教聊天面板”**：
*   **交互逻辑**：学生在查看生成的 `main.cpp` 时，可以用鼠标划选任意一行或一段代码（例如：STM32 的寄存器配置语句或库函数实例化）。
*   **AI 答疑**：选中的代码段会自动发送给内置 AI 助教，助教将采用“隐喻和生活化语言”进行科学解释（例如：*“这行代码相当于合上电闸，为 GPIOB 端口送电”*），避免生硬的技术堆砌。

### 3. 可视化电气防错“接线小游戏”（交互式拖拽配线）
将纯展示的 CAD 拓扑图扩展为可交互的教学仿真关卡：
*   **交互逻辑**：
    1.  教师可选择开启“手动接线挑战”模式。
    2.  系统随机隐藏 CAD 接线图中的几根关键线缆（如：I2C SDA/SCL、蜂鸣器控制信号线等）。
    3.  学生需要使用鼠标点击传感器引脚，并拖拽引脚线线缆，连线到主控板对应的 GPIO 或电源 Pin 脚上。
*   **仿真检验**：
    *   如果连线正确，系统触发连线咬合动画与成功音效；
    *   如果连线错误（例如电源极性接反导致 VCC 与 GND 短路，或者 5V 设备误接了不支持电平转换的 GPIO），系统将播放“烧毁警告”仿真动画，并提示错误成因。

### 4. 教师端“数据驱动教学评价”学情看板
为了服务教师的“数字评估与学情诊断”素养，在管理员/教师后台中新增教学评价仪表盘：
*   **数据采集**：统计学生使用该软件进行设计时的连线数据。
*   **指标分析**：
    *   **高频连线错误热力图**：统计全班学生在哪类外设（如 I2C 屏幕或 PWM 电机）的接线上最容易发生 ERC 引脚碰撞或电压越载错误，辅助教师发现教学盲点。
    *   **创客选题偏好图**：统计学生最喜欢设计的物联网场景类别（智能家居、环境监测等）及最爱选用的传感器，用于个性化辅导。

### 5. 双脑大模型对比学习（AI Literacy 批判性思考）
在方案优化页面，支持学生开启**“双模型评测模式”**：
*   **对比展示**：系统采用左右分栏，并排比对 Gemini 3.5 和 DeepSeek 针对同一份原始需求生成的代码实现结构。
*   **引导性提问**：系统在对比界面下方弹出 AI 批判性思考问题（例如：*“Gemini 使用了库函数实例化，而 DeepSeek 编写了原生底层寄存器配置，它们各有什么优缺点？哪个更容易移植？”*）。
*   **教学意义**：培养学生的批判性思维，避免盲目迷信单一 AI 的生成结果。
