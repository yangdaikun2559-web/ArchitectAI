# 《ArchitectAI 物联网智能生成与编译平台》教学课件设计与开发计划书

**项目名称**：ArchitectAI 物联网智能生成与编译平台  
**申报项目**：2026年四川省教师数字素养提升实践活动 - 常规项目（课件类）  
**适用组别**：中等职业教育组 / 高等教育组 / 基础教育组（创客与科学教育）  
**适用课程**：《物联网技术导论》、《单片机接口技术》、《嵌入式系统开发》、《Arduino创意编程》  

---

## 一、 项目背景与教学痛点分析

在传统的物联网与硬件开发实验教学中，学生和教师面临以下核心痛点：

1.  **“软硬割裂”的学习门槛高**：学生需要同时掌握复杂的 C++ 代码编写（嵌入式驱动）与电路拓扑连线。对于初学者，错别字、漏掉分号、引脚强占冲突常导致编译失败或硬件烧毁，极易挫伤学习积极性。
2.  **教学指导“黑盒化”，反馈滞后**：教师在实验课上需要逐一为几十名学生排查“为什么我的灯不亮”或“为什么编译报错”。由于缺乏可视化的电气安全检测与实时代码对齐工具，课堂效率低下。
3.  **缺乏分层差异化教学载体**：班级学生底子不同，统一的“喂入式”教学无法满足“学困生”的脚手架支持和“学优生”的自主创客探究。
4.  **AI 素养流于理论，缺乏实践场景**：虽然人工智能时代要求培养学生的“人机协同素养”，但日常课堂中缺乏让学生与大模型协同进行工程设计的具体课件载体。

## 二、 课件教学设计理念

本课件秉承 **“设计即代码 (Design as Code)”** 与 **“探究式做中学 (Inquiry-based Learning by Doing)”** 的理念，将大语言模型（LLM）引入硬件实验教学，通过以下三条支柱建立教学策略：

```
                    【探究式物联网学习支柱】
                     
       [ 支柱 1: 教学脚手架 ]  -->  经典实验库引导 + 分层扣空任务
       [ 支柱 2: 可视化安全反馈 ] -->  电气 ERC 检测报告 + 仿真接线游戏
       [ 支柱 3: 人机协同素养 ]  -->  AI 智能代码助教 + 双脑生成比对
```

1.  **教学脚手架机制 (Scaffolding)**：通过“经典实验库”和“Novice-Explorer-Expert”代码分层生成，为不同水平学生提供个性化学习支持。
2.  **可视化安全与即时反馈 (Visualized Safety & Instant Feedback)**：通过可视化电气规则检测（ERC）与仿真接线，实现“所接即所得，错接必有报”，培养安全规范意识。
3.  **人机协同与批判性思维 (Human-AI Collaboration & Critical Thinking)**：引入双大模型（Gemini / DeepSeek）代码比对与 AI 代码翻译，培养学生理解算法、质疑 AI 的数字时代高阶思维。

## 三、 课件功能模块规划

课件规划分为以下四大核心功能板块：

### 1. 探究式实验引导模块 (Inquiry & Scaffold Layer)
*   **经典教学实验库**：预置“智能路灯控制”、“温湿度警备”、“智能防盗门禁”等大纲案例，提供“一键导入”作为探究的起点。
*   **Novice-Explorer-Expert 分层模式**：
    *   *初学 (Novice)*：生成完整固件并附带保姆级中英文注释，辅助代码阅读。
    *   *探究 (Explorer)*：故意“扣空”控制语句，让学生结合硬件连线完成代码填空，检测逻辑掌握情况。
    *   *创客 (Expert)*：生成工业级模块化结构，适合自由创作。

### 2. 互动拓扑与百科模块 (Interactive CAD & Encyclopedia)
*   **元器件百科卡片**：在 CAD 接线图中点击任何传感器，弹出卡片展示其科学工作原理（如光敏电阻阻值变化）、引脚功能、适用电压。
*   **交互式连线微调**：允许学生直接在 CAD 图中拉拽、修改导线连接，前端实时重新计算并自动改写 C++ 代码中的 `pins.h` 引脚定义宏。

### 3. 安全检测与仿真预审模块 (ERC & GCC Simulation)
*   **可视化 ERC 检测专家**：对短路、引脚冲突、 Strapping 引脚误用、 5V/3.3V 电平过载进行实时黄色/红色警示，给出分压或隔离的排错指导。
*   **在线编译预审 (GCC Dry-run)**：在下载前提供网页端命令行编译日志仿真，检查缺失依赖并输出友好错误诊断。
*   **拖拽连线闯关小游戏**：针对关键物理连线，学生进行拖拽操作，若正负极反接触发“元件烧毁”仿真动效，加深安全印象。

### 4. 教师评价与学情看板 (Teacher Assessment & Analytics)
*   **高频错误分析**：统计并汇聚全班学生发生率最高的连线错误（如 I2C 总线碰撞或 VCC-GND 短路），用热力图展示给教师。
*   **选题偏好统计**：展示学生对传感器选型与物联网场景的应用偏好，支撑个性化教学评估与学情诊断。

## 四、 课堂教学应用场景设计

本课件在实际物联网教学中的推荐应用模式为 **“四步探究教学法”**：

```
[ 步骤 1: 创设情境 ] --> 学生提出物联网需求，利用 AI 优化成方案
        |
[ 步骤 2: 虚拟连线 ] --> 在 CAD 面板自主微调接线，查看 ERC 安全诊断报告
        |
[ 步骤 3: 逻辑探究 ] --> 切换到“扣空模式”，阅读百科卡片，完成代码填空
        |
[ 步骤 4: 实物烧录 ] --> 在线模拟编译无误后，下载 ZIP 包，在实物上组装并运行
```

1.  **创设情境与方案生成（15分钟）**：学生输入自己的物联网构想（如“做一个宿舍指纹锁”），平台大模型快速细化为需求提示词并推荐器件。
2.  **物理配线与安全审查（15分钟）**：学生根据 CAD 图进行引脚微调，阅读“电气安全检测报告”学习防错防烧毁知识。
3.  **代码阅读与逻辑探究（25分钟）**：学生进入“探究模式”，利用“AI代码助教”翻译难懂的寄存器初始化语句，并补全被 AI 扣空的关键控制代码。
4.  **云编译预审与硬件落地（25分钟）**：通过“在线 GCC 预审控制台”干跑通过后，下载 ZIP 代码包导入实物开发板运行，教师通过“学情看板”开展多维表现性评价。

## 五、 开发路线图与工时预估 (Roadmap)

项目整体开发计划共计 **11 天**（预计 2 周内完成），分为六个迭代周期：

*   **S1：基础框架建立（1.5 天）**：完成“经典实验库”页面与一键导入模板配置。
*   **S2：百科与检测可视化（1.0 天）**：完成元器件科普卡片组件与 ERC 诊断报告前端面板。
*   **S3：编译仿真器开发（1.5 天）**：开发模拟 GCC 命令行输出控制台，编写基本语法静态过滤器。
*   **S4：AI 助教与分层机制（2.0 天）**：对接 AI 模型接口，实现代码行选翻译与代码主动扣空生成逻辑。
*   **S5：CAD 双向绑定（2.0 天）**：实现 CAD 图引脚下拉更改与 C++ `pins.h` 的双向更新机制。
*   **S6：游戏关卡与教师看板（3.0 天）**：完成拖拽接线防错游戏与教师学情统计看板，项目封包测试。

## 六、 预期成果与课件推广价值

1.  **降低教学成本与硬件折损**：虚拟 ERC 与接线闯关可过滤掉 90% 以上的学生物理连线低级错误，极大降低实验室元器件的烧毁和折损率。
2.  **全面提升教师数字素养**：为教师提供“大模型双脑协同+学情评价看板”的先进教学手段，为教师的数字化转型提供可复制、可推广的实践课例支撑。
3.  **培养学生人工智能思维**：不仅是消费 AI 生成的内容，更能通过代码比对、分层挑战，让学生理解技术局限性，成长为未来的科技创造者。
