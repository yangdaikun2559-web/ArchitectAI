# ArchitectAI 课件赛道参赛可行性与功能修改综合报告

## 一、结论摘要

ArchitectAI 可以参加“2026 年四川省教师数字素养提升实践活动”中的“常规项目 - 课件”赛道，而且这是当前项目最匹配的申报方向。

建议参赛定位不要写成“AI 物联网生成平台”或“代码生成工具”，而应明确包装为：

**面向物联网与单片机课程的 AI 辅助探究式学习课件**

推荐作品名称：

**ArchitectAI：基于 STM32 的物联网探究式学习与仿真实验课件**

当前项目已经具备较强的技术亮点，包括 AI 需求优化、硬件方案生成、CAD 接线图、BOM 清单、代码生成、工程下载、课堂/班级/学习任务等能力。主要短板不是“技术不够”，而是还需要把功能呈现改得更像“课件”，让评委明确看到教学目标、教学过程、学生任务、教师评价和课堂应用价值。

## 二、评审依据

本报告依据以下文件整理：

- 《四川省教育厅关于举办 2026 年四川省教师数字素养提升实践活动的通知》
- 《附件 1：2026 年四川省教师数字素养提升实践活动指南（常规项目）》

指南中“课件”的关键定义是：基于数字化、网络化、智能化信息技术，围绕教学内容、学情、目标、过程、方法与评价进行整体设计并制作完成的应用软件。各类教学软件、学生自主学习软件、教学评价软件、仿真实验软件等均可报送。

课件类报送材料包括：

- 作品登记表签字扫描件，PDF 格式。
- 课件本体，PPT/PPTX/ENBX，其他格式使用压缩包上传。
- 课件演示视频，MP4 格式。
- 相关设计说明，Word 文档格式。
- 相关材料，ZIP 压缩包格式。
- 总大小建议不超过 700MB。
- 如需其他软件运行，应在附表中注明软件名称、下载网址或分享链接。
- 演示视频应清晰演示课件实际运行过程，并配同步音频讲解；画面尺寸 1280x720 以上，播放时间一般不超过 10 分钟。

常规项目按基础教育组、中等职业教育组、高等教育组分别设置奖项。具体组别按作品第一作者所在单位划分。

## 三、是否可以参赛

### 1. 赛道匹配判断

可以参赛，建议申报：

**常规项目 - 课件**

不建议申报“微课”，因为本项目不是单一有声教学视频，而是一个可运行、可交互、可支持学生自主学习和教师评价的应用软件。

也不建议优先申报“融合创新应用教学案例”或“信息化教学课程案例”，除非已经有真实课堂实录、教学应用数据、学生成果和推广材料。当前项目更像“课件软件本体”，因此课件赛道最合适。

### 2. 当前项目与课件定义的对应关系

| 指南要求 | ArchitectAI 当前对应情况 | 判断 |
|---|---|---|
| 数字化、网络化、智能化信息技术 | React Web 应用、AI 需求优化、AI 代码生成、班级数据管理 | 符合 |
| 围绕教学内容设计 | 已聚焦物联网、STM32/ESP32、传感器、接线、代码 | 基本符合 |
| 围绕学情设计 | 已有工程学习脚手架和学习任务，但入口和表达还需强化 | 需增强 |
| 围绕教学目标设计 | 文档已有目标，软件中目标呈现还不够集中 | 需增强 |
| 围绕教学过程设计 | 有需求输入、方案生成、接线图、代码、下载、课堂提交流程 | 符合但需重组 |
| 围绕方法与评价设计 | 有班级、学习任务、教师反馈能力 | 符合但需突出 |
| 应用软件/仿真实验软件 | 项目属于可运行网页课件和物联网仿真实验辅助软件 | 符合 |
| 易安装、运行、卸载 | 目前仍偏开发项目，需要参赛运行包和说明 | 需补齐 |
| 演示视频 | 已有脚本基础，但需按真实页面录制 | 需完成 |

综合判断：当前项目具备参赛资格，且有较好的获奖潜力；但必须完成“课件化呈现”和“提交材料完整化”。

## 四、当前优势

### 1. 技术亮点鲜明

项目不是普通 PPT 课件，而是可交互的智能教学软件，能体现“数字素养提升”的主题。AI 生成、硬件规则校验、CAD 接线、代码导出等能力具有辨识度。

### 2. 与物联网教学痛点高度相关

物联网和单片机实验中，学生常见问题是不会拆解需求、不会选型、不会接线、代码与硬件脱节、安全意识不足。ArchitectAI 正好围绕这些痛点设计，可以形成清晰的教学价值叙事。

### 3. 已具备课堂功能雏形

仓库中已经有班级、学生提交、教师反馈、学习任务、工程脚手架等功能。只要把入口、文案和演示流程收拢，就能呈现出“学生学、教师评、课后留痕”的完整课件形态。

### 4. 已有 STM32 参赛方向基础

项目当前已支持 STM32F103C8T6、Keil 工程模板、标准外设库、DHT22/MQ-2/OLED/蜂鸣器/ESP8266 等典型硬件组合，适合包装为中职或高职单片机/物联网课程案例。

## 五、主要不足与风险

### 1. 产品表达偏“平台”，课件表达不足

当前名称、文案和流程容易让评委理解成“AI 工程生成器”。参赛时应强调“它服务一节课/一组教学任务”，而不是强调“它能生成很多项目”。

建议把主叙事改为：

学生从生活情境出发，通过 AI 辅助完成需求建模、器件选型、虚拟接线、安全判断、代码理解和工程导出，教师基于过程数据开展评价。

### 2. 教学目标、学生任务和评价量表还不够显性

指南对课件的推荐指标首先看教学目标、对象、策略、界面、交互、帮助文档。当前项目有学习任务功能，但需要在主流程中更醒目地呈现。

### 3. 中文编码与材料一致性需要清理

仓库中部分文档和 UI 字符曾出现乱码或编码损坏。如果最终提交材料中出现乱码，会严重影响评审观感。参赛前必须统一检查：

- UI 中文文案。
- README 和运行说明。
- 设计说明 Word。
- 作品登记表中的 300 字简介。
- 演示视频字幕和画面文字。

### 4. 提交形态还不完整

目前是开发项目形态，参赛需要“评委能打开、能理解、能复现”的作品包。需要补齐可运行包、安装使用说明、演示账号、演示视频、设计说明和相关材料 ZIP。

### 5. 真实教学证据不足

如果想冲更高奖项，最好补充课堂应用或模拟教学证据，例如学生任务单、评价量表、试用截图、教师反馈样例、学生作品样例。没有真实课堂数据也可以先做“模拟课堂流程”和“样例学生提交记录”。

## 六、功能修改建议

下面按优先级列出。你前面说“代码优化放后面”，因此这里不建议先拆架构，而是先改功能呈现。

### P0：必须完成，直接影响能否像课件

#### 1. 增加“参赛演示模式”入口

在首页或侧边栏增加一个醒目的入口：

**进入课件演示：STM32 智能温室环境监测**

点击后应自动加载固定课例，并直接进入完整学习流程，而不是让评委从空白输入开始。

课例建议固定为：

**基于 STM32 的智能温室环境监测与报警系统**

硬件组合：

- STM32F103C8T6 Blue Pill
- DHT22 温湿度传感器
- MQ-2 气体/烟雾传感器
- OLED SSD1306 显示屏
- 有源蜂鸣器
- ESP8266 WiFi 模块

#### 2. 增强“课件化学习流程”页面

把当前生成结果页之外，增加或强化一个“学习流程”视图，按课堂任务组织：

1. 创设情境：温室需要监测温湿度和空气质量。
2. 需求建模：学生输入或选择需求，AI 转成工程方案。
3. 器件选型：解释每个元器件的作用。
4. 虚拟接线：查看 CAD 接线图和引脚表。
5. 安全判断：识别 5V/3.3V、电平保护、共地、驱动风险。
6. 代码探究：对照 `pins.h` 和 `main.c` 理解采集、显示、判断、报警逻辑。
7. 工程导出：下载 Keil/PlatformIO 工程包。
8. 教师评价：提交任务，教师查看反馈。

#### 3. 在结果页突出“电气安全检测报告”

这是最能体现课件价值和创新性的功能之一。建议在 CAD/BOM 附近增加独立模块：

**电气安全与接线规范检测**

展示内容：

- 引脚冲突检测。
- I2C 总线共享说明。
- 5V 模块接入 3.3V MCU 的风险提示。
- 蜂鸣器、继电器、电机等执行器的驱动建议。
- 共地提醒。
- 修改建议。

这比单纯展示代码更贴近教学评分中的“技术运用有效”和“实际教学可用”。

#### 4. 增加“代码教学解读”侧栏

在代码查看页中增加对学生友好的解释卡片，而不是只展示源码：

- `pins.h`：硬件接线与代码宏定义的对应关系。
- `GPIO_Configuration`：输入输出引脚配置。
- `ADC_Configuration`：MQ-2 模拟量采集。
- `I2C/OLED`：显示屏通信。
- `DHT22_ReadData`：温湿度读取。
- `Buzzer_Set`：报警控制。
- `TEMP_THRESHOLD/GAS_THRESHOLD`：阈值判断。

这能把“代码生成工具”转化为“代码理解课件”。

#### 5. 准备可提交运行包和说明

作品本体建议以 ZIP 提交，包内至少包含：

- 可运行项目或构建产物。
- `README_运行说明.md`。
- 环境要求。
- 启动命令。
- 演示账号。
- 参赛演示入口说明。
- 常见问题。
- 如果依赖在线 AI 服务，应说明离线演示备用方案。

### P1：强烈建议完成，用于提升获奖竞争力

#### 1. 学生任务单功能或导出材料

围绕 8 个学习步骤生成学生任务单：

#### 2. 教师评价量表

评价维度建议：

| 维度 | 评价内容 |
|---|---|
| 需求分析 | 能否把生活问题拆成系统功能 |

#### 3. 教师端“学情看板”更贴近评审

## 八、作品 300 字简介建议

ArchitectAI 是一款面向物联网与单片机课程的 AI 辅助探究式学习课件。课件以“基于 STM32 的智能温室环境监测与报警系统”为主任务，引导学生从生活需求出发，完成需求分析、器件选型、虚拟接线、安全判断、代码理解和工程导出。系统集成 AI 方案生成、CAD 接线图、BOM 清单、元器件百科、电气安全检测、代码教学解读和教师过程性评价等功能，帮助学生理解主控、传感器、显示、报警和通信模块之间的工程关系。课件将传统物联网实验中易错的接线、代码和硬件安全问题转化为可视化、可反馈、可评价的学习过程，既降低实验入门难度，又培养学生的人机协同能力、工程思维和数字素养，适用于中职、高职及相关创客课程教学。

## 九、安装运行说明 300 字建议

本课件为网页应用软件，推荐使用 Windows 10/11 系统和 Chrome 或 Edge 浏览器运行。解压作品压缩包后，按照 `README_运行说明.md` 安装 Node.js 依赖并启动服务；如提交版本提供在线演示地址，可直接通过浏览器访问。演示账号和密码见运行说明。进入系统后点击“课件演示：STM32 智能温室环境监测”即可加载参赛课例，依次体验需求建模、AI 方案生成、虚拟接线、电气安全检测、代码解读、工程下载和教师评价流程。若本地未配置 AI 接口密钥，可使用内置演示数据完成完整课件演示。
